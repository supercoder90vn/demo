VIII. Validation: novalidate, $invalid, $valid, $pristine, $dirty, $error

1.
a. novalidate
< form novalidate> => avoid validating by HTML5,
b.$invalid, $valid
only be able to submit if it is valid
c. $pristine, $dirty
never show error if the user has not interacted with the field yet ($pristine or not $dirty)
2. $error
$error is an object containig informations about error of inputs of form with fields corresponding to name of them

use console.log($scope.myform) to see more information

3.some validations
a.required

< form name="myForm"> < input name="username" required> => myForm.username.$error.required

Username: Username is required.

b. type="email"

< form name="myForm"> < input name="email" type="email"> => myForm.email.$error.email

Email: Email is required. Invalid email address.
